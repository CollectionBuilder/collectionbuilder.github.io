CollectionBuilder's basic metadata profile is based on Orbis Cascade Alliance Dublin Core Best Practices Guidelines, which are designed to enable interoperability with aggregators, particularly DPLA.

This template contains the basic set of fields that CollectionBuilder needs to work properly, but users are welcome to add additional fields as needed to contextualize their collections.

Metadata Workflow

The templates above have all been created in Google Sheets. We recommend copying these versions of the metadata template into your own Google Sheets account to work on it. Follow the example record in these templates to create your metadata, or view our CollectionBuilder Data Dictionary for more specific guidelines about each field.

Next Steps:

1. Download Your CSV

  • Once you’ve finished creating your metadata in Google Sheets, click “File” and select “Download as Comma-separated values.”
  • Locate the metadata csv you just downloaded on your computer. Without opening it, name this file using all lowercase letters and no spaces (i.e. collection-metadata.csv). Drop the file into the _data folder.

2. Configure Your Metadata Display

  • Locate metadata-config.csv in the _data folder.
  • Edit this file to choose the order and display name for the metadata fields as they appear on the collection's item pages and visualizations.
  • Fields not included in the metadata-config.csv will not be displayed on the site.
